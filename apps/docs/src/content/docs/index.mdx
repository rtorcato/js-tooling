## Why this exists

Most tooling libraries give you one piece — just TypeScript configs, or just an ESLint preset. **js-tooling** covers the entire lifecycle: TypeScript, Biome/ESLint, Vitest/Jest, Commitlint, Husky, Semantic Release, and GitHub Actions CI, all wired together and validated against each other.

The interactive `setup` wizard scaffolds all of it in one shot for a new project, and `doctor` checks an existing project for drift.

```bash
npx @rtorcato/js-tooling setup
```
